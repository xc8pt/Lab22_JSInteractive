# Калькулятор на JavaScript

## 1. Описание проекта

Проект представляет собой простой веб-калькулятор, реализованный на чистом JavaScript (ES6) с использованием классов. Калькулятор позволяет выполнять базовые арифметические операции (`+`, `-`, `*`, `/`), обрабатывает ввод с кнопок интерфейса, предотвращает некорректные выражения и выводит результат или сообщение об ошибке.

Проект подходит для изучения работы с DOM, обработки событий, валидации ввода и безопасного вычисления выражений в браузерной среде.

## 2. Функциональность

### Какие операции поддерживает?
- Сложение (`+`)
- Вычитание (`-`)
- Умножение (`*`)
- Деление (`/`)
- Ввод целых и десятичных чисел 
- Очистка всего ввода (кнопка `C`)
- Вычисление результата (кнопка `=`)

### Какие есть ограничения?
- Не поддерживаются скобки, унарные операторы (кроме унарного минуса в начале выражения), операции с плавающей точкой выполняются стандартным способом JavaScript.
- Максимальная длина выражения не ограничена, но слишком длинные строки могут визуально выходить за пределы поля ввода.
- Деление на ноль даёт `Infinity`, что перехватывается и заменяется сообщением об ошибке.
- Не поддерживается ввод с клавиатуры - только клики по кнопкам.
- Нельзя вводить несколько операторов подряд (например, `++` или `*/`), а также завершать выражение оператором.

### Как обрабатываются ошибки?
- При попытке вычислить некорректное выражение (например, `2++3`, `5/0`, пустая строка) вызывается метод `showError()`, который:
  - Выводит в поле ввода текст `"Ошибка"`.
  - Устанавливает флаг `hasError = true`.
  - Очищает внутреннюю строку `expression`.
- Если флаг ошибки активен, любое нажатие кнопки (кроме `C`) сначала автоматически очищает калькулятор (вызывается `clear()`), чтобы пользователь мог начать новый ввод.
- В методе `safeEvaluate()` дополнительно проверяется, является ли результат конечным числом (`isFinite`). Если нет - генерируется ошибка.
- Валидация выражения выполняется перед вычислением: проверяется наличие двух и более операторов подряд, а также завершающего оператора.

## 3. Архитектура

### Структура файлов
Проект состоит из двух основных файлов:

- **`calculator.js`** - содержит класс `Calculator`, всю логику обработки ввода, валидации, вычислений и обновления интерфейса.
- **`app.js`** (или основной скрипт) - инициализирует экземпляр класса `Calculator` и запускает его метод `init()` после загрузки DOM.

Также присутствует HTML-файл (в коде не показан), который должен содержать:
- Элемент `<input id="display" readonly>` для отображения ввода и результата.
- Набор кнопок с текстом: цифры, операторы (`+`, `-`, `*`, `/`), точка (`.`), `C`, `=`.

### Описание класса `Calculator`

| Свойство / Метод | Тип | Назначение |
|----------------|------|-------------|
| `display` | DOM-элемент | Поле ввода, куда выводится выражение или результат. |
| `expression` | строка | Хранит текущее математическое выражение. |
| `hasError` | boolean | Флаг состояния ошибки (true - показана ошибка). |
| `init()` | метод | Навешивает обработчики событий на все кнопки, вызывает начальное обновление дисплея. |
| `handleButtonClick(value)` | метод | Маршрутизирует нажатия: `C` → очистка, `=` → вычисление, остальное -> добавление в выражение. |
| `addToExpression(value)` | метод | Добавляет символ к `expression` с учётом правил: запрет двойных операторов, точки, ведущего оператора (кроме минуса). |
| `calculate()` | метод | Вызывает безопасное вычисление, обрабатывает результат или ошибку. |
| `safeEvaluate(expr)` | метод | Выполняет `eval(expr)` после предварительной проверки `isValidExpression`. |
| `isValidExpression(expr)` | метод | Проверяет выражение на наличие двух операторов подряд или оператора в конце. |
| `clear()` | метод | Сбрасывает `expression` и `hasError`, обновляет дисплей. |
| `updateDisplay()` | метод | Выводит `expression` (или `"0"`, если пусто) в поле ввода, если нет ошибки. |
| `showError(message)` | метод | Устанавливает текст ошибки, включает флаг `hasError`, очищает выражение. |
| `isOperator(char)` | метод | Возвращает `true`, если символ является оператором. |
| `isLastCharOperator()` | метод | Проверяет, заканчивается ли текущее выражение оператором. |
| `hasDuplicateDot()` | метод | Определяет, содержит ли последнее введённое число уже точку (запрещает повторную точку в том же числе). |

## 4. Логика работы

### Последовательность действий при нажатии кнопок

1. **Кнопка цифры или десятичной точки**  
   - Символ добавляется в конец строки `expression` (с проверками на корректность).  
   - Обновляется дисплей.

2. **Кнопка оператора (`+`, `-`, `*`, `/`)**  
   - Если выражение пустое и оператор не является минусом (`-`) - добавление блокируется.  
   - Если последний символ уже оператор - заменяет его на новый (не допускает `++` или `*-`).  
   - В остальных случаях оператор добавляется.

3. **Кнопка `C` (очистка)**  
   - Сбрасывает `expression = ""`, `hasError = false`, дисплей показывает `"0"`.

4. **Кнопка `=`**  
   - Запускает вычисление. Если выражение валидно - результат замещает `expression` и выводится на дисплей.  
   - При ошибке  дисплей показывает `"Ошибка"`, флаг `hasError = true`, выражение очищается.

### Как работает валидация ввода?

Валидация происходит на двух уровнях:

- **Во время ввода** (метод `addToExpression`):
  - Запрет двух операторов подряд (заменяет последний оператор новым).
  - Запрет ведущего оператора, если это не `-` (чтобы можно было ввести отрицательное число).
  - Запрет повторной точки в одном числе (метод `hasDuplicateDot`).
- **Перед вычислением** (метод `isValidExpression`):
  - Проверяет, что нет двух и более операторов подряд (`[+\-*/]{2,}`).
  - Проверяет, что выражение не заканчивается оператором.
- **Во время вычисления** (`safeEvaluate`):
  - Дополнительная проверка результата на конечность (`isFinite`), чтобы отловить деление на ноль и `Infinity`.

### Как происходит вычисление?

1. Пользователь нажимает `=`.
2. Вызывается `calculate()`.
3. Проверяются условия:  
   - Если `expression` пустая или `hasError == true` - выход.
4. Выполняется `safeEvaluate(expression)`:
   - Сначала вызывается `isValidExpression()` для базовой проверки.
   - Затем используется встроенная функция `eval(expr)`.  
     *Примечание:* в учебном проекте `eval` допустим, так как выражение полностью контролируется кнопками. В реальном приложении следовало бы заменить на более безопасный парсер.
5. Полученный результат проверяется на `isFinite()`.
6. Если всё успешно:
   - `expression = String(result)`
   - `hasError = false`
   - Вызов `updateDisplay()` - показывает результат.
7. Если в любом месте возникает ошибка (исключение):
   - Вызывается `showError("Ошибка")` -> дисплей показывает `"Ошибка"`, `expression` очищается, `hasError = true`.
